How the Web Works

The web is a bunch of computers that communicate with each other. When a person goes to a web page like www.udacity.com, their computer sends request to a server using HTTP protocol via the internet. The server hosts the files and finds the appropriate HTML document. It then sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

Five important things to take away from this lesson:

1 - What a Web Page is

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

2 - How Coding Works

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote. For example, the computer might do this by making text like this: I'm learning to code! looks like this: I'm learning to code!

3 - Computers are Stupid
Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax").
For example, if you forget to close a b tag, the computer won't be able to figure out what you had intended to make bold. This "stupidity" can be very frustrating, but it also gives programmers incredible power: if you know how to talk to a computer than you can tell it do do anything you want.

4 - Programmers Can't Remember Everything
There are too many details to keep everything in your head. And that's okay. If you forget how to make text italic in HTML, you can always just look it up.

5 - Basic HTML Vocabulary
You will be using HTML in the next few lessons, so it will be helpful if you're comfortable with the jargon.

Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag (p for example) and a closing tag, (/p). Some tags (called "void" tags) do not require a closing tag (like the br tag).

Element: An HTML element refers to everything within a set of opening and closing tags.

Attribute: This is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write a href="www.udacity.com”

Benefits of Avoiding Repetition

Repetition is doing the same thing over and over again. When the code needs to be changed, it is more prone to errors if those changes need to be replicated in many different places. If the change isn't made in every place errors can arise. Programmers should avoid repetition whenever possible.

In CSS, using Inheritance properties for specified elements and its descendants helps reduce the need for repetition. Inheritance allows programmers to write less CSS by preventing certain properties from being repeated over and over again.

Why Don't We Use Natural Language to Program Computers?

Ambiquity - natural languages are inherently ambiguous, different people interpret it in different ways. The computer needs to understand the language and interpret it the same way each time. Example - definition of bi-weekly has two different meanings (every other week vs twice a week). If a computer program needed to manage payroll, the term biweekly would need to be clearly defined.

Verbosity - Natural language needs a lot of words to precisely explain things. We need to keep it as streamlined and simple as possible for the computer.

Grammar - Programming language use grammar just like human languages. It specifies what is correct vs incorrect. In spoken language, we can often make sense of what someone istrying to say, even if the grammar is incorrect. Computers cannot do this. The code needs to be precise and correct (according to the Python interpreter) or the code won't work. Print = Python code to print the result of the expression after it.

SyntaxError - the grammar is not in the programming languageCode must be written with specific syntax. A small type can lead to a big error.
